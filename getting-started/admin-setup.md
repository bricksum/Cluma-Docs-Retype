# 관리자 초기 설정

Cluma를 처음 배포한 후 사용자들이 컨테이너를 생성할 수 있도록 준비하는 절차입니다.

기본 관리자 계정: `admin@bricksum.com` / `admin`

!!!warning 기본 비밀번호 변경
최초 로그인 후 반드시 기본 비밀번호를 변경하세요.
!!!

---

## 1단계 — 그룹 생성 및 리소스 한도 설정

사용자들이 사용할 그룹을 만들고 리소스 한도를 설정합니다. 사용자는 그룹에 배정된 한도 안에서만 컨테이너와 스토리지를 생성할 수 있습니다.

1. 관리자 메뉴 → **그룹 관리** → **그룹 생성**
2. 그룹 이름을 입력하고 리소스 한도를 설정합니다.

| 항목 | 설명 |
|------|------|
| CPU | 그룹 전체가 사용할 수 있는 최대 CPU 코어 수 |
| 메모리 | 그룹 전체 최대 메모리 (GB) |
| 디바이스 | 가속기 유형별 최대 수량 |
| 스토리지 | 그룹 전체 최대 스토리지 용량 (GB) |

자세한 내용은 [그룹 관리](/admin-guide/group-management.md)를 참고합니다.

---

## 2단계 — 사용자 생성 및 그룹 배정

사용자 계정을 만들고 그룹에 배정합니다.

1. 관리자 메뉴 → **사용자 관리** → **사용자 생성**
2. 이메일, 이름, 비밀번호를 입력하고 생성합니다.
3. 생성된 사용자 상세 페이지에서 그룹을 배정합니다.

자세한 내용은 [사용자 관리](/admin-guide/user-management.md)를 참고합니다.

---

## 3단계 — 레포지토리 등록 (선택)

컨테이너 생성 시 사용할 Private Registry를 등록합니다. 등록하지 않으면 사용자는 퍼블릭 이미지 주소를 직접 입력해야 합니다.

1. 관리자 메뉴 → **레포지토리 관리** → **레포지토리 등록**
2. 이름과 레지스트리 주소(URL)를 입력합니다.

자세한 내용은 [레포지토리 관리](/admin-guide/registry-management.md)를 참고합니다.

---

## 4단계 — 클러스터 상태 확인

노드와 리소스가 정상적으로 연결되어 있는지 확인합니다.

- 관리자 메뉴 → **클러스터 대시보드**: 노드 상태(Ready/Drained), Pod 현황 확인
- 관리자 메뉴 → **관리자 대시보드**: 클러스터 전체 CPU/메모리/GPU/스토리지 현황 확인

---

## 준비 완료

위 단계가 완료되면 사용자들이 [사용자 빠른 시작](./quickstart.md)을 따라 컨테이너를 생성할 수 있습니다.
