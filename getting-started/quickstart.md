# 사용자 빠른 시작

처음 Cluma를 사용하는 경우 아래 순서로 진행합니다.

---

## 1단계 — 로그인

관리자로부터 받은 계정 정보로 로그인합니다.

로그인 후 사이드바에서 소속 그룹이 표시되는지 확인합니다. 그룹이 없는 경우 관리자에게 그룹 배정을 요청합니다.

---

## 2단계 — 잔여 리소스 확인

상단 헤더에서 현재 그룹의 CPU / 메모리 / GPU / 스토리지 잔여량을 확인합니다.

컨테이너와 스토리지는 그룹의 리소스 한도 안에서만 생성할 수 있습니다. 잔여 리소스가 부족한 경우 관리자에게 한도 증가를 요청합니다.

---

## 3단계 — 스토리지 생성

학습 데이터, 모델 파일 등 컨테이너가 삭제되어도 유지해야 할 데이터는 스토리지에 저장합니다.

1. 사이드바 → **스토리지** → **스토리지 생성**
2. 이름, 용량, 스토리지 유형을 입력하고 생성합니다.

자세한 내용은 [스토리지 관리](/user-guide/storage.md)를 참고합니다.

---

## 4단계 — SSH Key 등록 (선택)

비밀번호 없이 SSH로 컨테이너에 접속하려면 SSH Key를 미리 등록합니다.

1. 우측 하단 프로필 메뉴 → **SSH Key 관리**
2. Key 이름을 입력하고 공개키를 직접 입력하거나 파일로 업로드합니다.

---

## 5단계 — 컨테이너 생성

1. 사이드바 → **컨테이너** → **새 컨테이너 만들기**
2. **고급 모드**에서 아래 항목을 설정합니다.

| 항목 | 설명 |
|------|------|
| 컨테이너 이름 | 식별 이름 |
| 이미지 | 레지스트리 탐색 또는 퍼블릭 주소 직접 입력 |
| CPU / 메모리 | 필요한 컴퓨팅 리소스 선택 |
| 가속기(GPU) | 필요한 GPU 수 선택 (0이면 CPU 전용) |
| 포트 포워딩 | SSH(22), Jupyter(8888), VSCode(8080) 등 |
| 스토리지 마운트 | 3단계에서 만든 스토리지 선택 및 마운트 경로 지정 |
| SSH Key | 4단계에서 등록한 Key 선택 |

자주 사용하는 설정은 **템플릿으로 저장**해두면 다음에 원클릭으로 빠르게 생성할 수 있습니다.

---

## 6단계 — 컨테이너 접속

컨테이너 상태가 **Running**이 되면 접속할 수 있습니다.

| 방법 | 설명 |
|------|------|
| Jupyter | 상세 페이지 → Jupyter 버튼 → 브라우저에서 JupyterLab 열기 |
| Terminal | 상세 페이지 → Terminal 버튼 → SSH 터미널 연결 |
| VSCode | 상세 페이지 → VSCode 버튼 → VSCode Desktop SSH Remote 연결 |
| SSH | 상세 페이지 → SSH 접속 명령어 복사 후 터미널에서 실행 |
