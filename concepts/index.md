# 핵심 개념

Cluma를 사용하기 전에 알아야 할 핵심 개념을 설명합니다.

---

## 유저 (User)

Cluma에 로그인하는 사용자 계정입니다. 모든 유저는 **관리자** 또는 **일반 사용자** 중 하나의 역할을 가집니다.

### 관리자 (Admin)

- 유저 생성 / 수정 / 삭제
- 그룹 생성 / 수정 / 삭제
- 유저를 그룹에 할당 또는 제거
- 전체 클러스터 리소스 모니터링
- 노드, 레지스트리 등 인프라 관리

### 일반 사용자 (Regular User)

- 소속 그룹의 리소스 범위 내에서 컨테이너 생성 및 관리
- 스토리지 생성 및 마운트
- 개인 대시보드에서 리소스 현황 확인
- 그룹 소속 변경 불가 (관리자만 가능)

---

## 그룹 (Group)

유저를 묶고 리소스 한도를 정의하는 단위입니다.

- 유저는 하나 이상의 그룹에 소속될 수 있습니다.
- 그룹마다 CPU, 메모리, GPU, 스토리지 한도를 설정합니다.
- 그룹과 1:1로 대응하는 **Kubernetes Namespace**가 생성됩니다.
- 유저는 소속 그룹의 리소스 한도 내에서만 컨테이너와 스토리지를 생성할 수 있습니다.
- 신규 유저는 `Default` 그룹에 자동으로 배정됩니다.

```
그룹 A (CPU: 32코어 / 메모리: 128GB / GPU: 4개)
├── 유저 1
├── 유저 2
└── 유저 3
```

## 컨테이너 (Container)

Cluma에서 GPU 개발 환경의 핵심 단위입니다.  
Kubernetes Deployment 기반으로 동작하며, 안정적인 워크로드 운영이 가능합니다.

### 컨테이너 구성 요소

| 항목 | 설명 |
|------|------|
| 이미지 | 컨테이너 실행에 사용할 Docker 이미지 |
| CPU / 메모리 | 할당할 컴퓨팅 리소스 |
| GPU | 할당할 GPU 수 (MIG 분할 포함) |
| 포트 포워딩 | SSH, Jupyter, VSCode 등 접속 포트 설정 |
| 스토리지 마운트 | PVC를 원하는 경로에 연결 |

### 리소스 할당 방식

- **Dynamic**: 요청한 리소스를 유연하게 할당 (공유 환경에 적합)
- **Static**: 고정 리소스를 전용으로 점유 (안정적인 학습 작업에 적합)

---

## 스토리지 (Storage)

데이터를 영구적으로 저장하고 컨테이너에 마운트할 수 있는 저장 공간입니다.  
Kubernetes PVC(Persistent Volume Claim) 기반으로 동작합니다.

### 스토리지 유형

| 유형 | 설명 |
|------|------|
| **개인 스토리지** | 본인만 접근 가능한 전용 저장 공간 |
| **그룹 공유 스토리지** | 동일 그룹 내 유저들이 함께 사용하는 저장 공간 |

### StorageClass

관리자가 등록한 StorageClass(NFS, Local 등)를 선택하여 스토리지를 생성합니다.

---

## 이미지 (Image)

컨테이너 실행에 사용할 Docker 이미지입니다.

- 관리자가 등록한 **Private Registry**에서 이미지를 탐색하고 선택할 수 있습니다.
- 퍼블릭 이미지 주소를 직접 입력할 수도 있습니다.
- 일반 사용자도 커스텀 이미지를 직접 등록하고 관리할 수 있습니다.

---

## GPU 분할 (MIG)

NVIDIA Multi-Instance GPU(MIG) 기술을 통해 하나의 GPU를 여러 워크로드에 분할하여 할당할 수 있습니다.  
GPU 자원을 효율적으로 활용할 때 사용합니다.
