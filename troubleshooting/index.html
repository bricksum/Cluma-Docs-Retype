트러블슈팅

Cluma 사용 중 자주 발생하는 문제와 해결 방법을 설명합니다.

컨테이너

컨테이너가 Pending 상태에서 전환되지 않는 경우

클러스터 GPU 리소스 부족

클러스터에 사용 가능한 GPU가 없는 경우입니다. 관리자 대시보드에서 GPU 사용 현황을 확인합니다. 다른 컨테이너가 종료되면 자동으로 스케줄링됩니다.

그룹 리소스 한도 초과

소속 그룹의 GPU 할당량을 초과한 경우입니다. 대시보드 상단에서 그룹 잔여 리소스를 확인하고 불필요한 컨테이너를 삭제합니다.

노드 비활성 상태

관리자에게 노드 활성화 상태 확인을 요청합니다.

컨테이너가 예기치 않게 중지된 경우

GPU 사용 시간 소진

유저 또는 그룹에 할당된 GPU 사용 시간이 소진되면 컨테이너가 자동으로 정지됩니다. 관리자에게 GPU 사용 시간 추가 할당을 요청합니다.

OOM (메모리 초과)

컨테이너에 할당된 메모리를 초과하면 강제 종료됩니다. 컨테이너 로그를 확인하고 메모리 할당량을 늘려 재생성합니다.

SSH 접속이 되지 않는 경우

아래 항목을 순서대로 확인합니다.

  • 컨테이너 상태가 Running인지 확인
  • SSH 포트 포워딩이 설정되어 있는지 확인
  • 접속 명령어의 포트 번호가 올바른지 확인
  • SSH Key를 사용하는 경우 Key 파일 경로가 올바른지 확인
ssh -p <포트번호> root@<노드IP>

# SSH Key 사용 시
ssh -i ~/.ssh/id_rsa -p <포트번호> root@<노드IP>

JupyterLab / VSCode 접속이 되지 않는 경우

  • 컨테이너 상태가 Running인지 확인
  • 해당 포트 포워딩이 설정되어 있는지 확인
  • 브라우저 주소의 포트 번호가 올바른지 확인
  • SSH로 접속하여 컨테이너 내부에서 해당 서비스가 실행 중인지 확인

스토리지

스토리지 마운트가 되지 않는 경우

  • 스토리지가 정상적으로 생성된 상태인지 확인
  • 마운트 경로가 올바르게 입력되었는지 확인
  • 그룹의 스토리지 용량 한도를 초과하지 않았는지 확인

스토리지 용량이 부족한 경우

기존 스토리지의 용량을 확장할 수 있습니다. 스토리지 목록에서 해당 스토리지를 선택한 후 용량 확장을 클릭합니다.

그룹의 스토리지 총 한도 내에서만 확장할 수 있습니다. 한도 초과가 필요한 경우 관리자에게 문의합니다.

이미지

이미지를 불러올 수 없는 경우

  • 레지스트리가 관리자에 의해 정상 등록되어 있는지 확인
  • 이미지 이름과 태그가 정확한지 확인
  • 퍼블릭 이미지 주소를 직접 입력한 경우 클러스터에서 외부 네트워크 접근이 가능한지 확인

문제가 해결되지 않는 경우

관리자에게 아래 정보를 함께 전달합니다.

  • 발생한 증상 및 시간
  • 컨테이너 이름 및 상태
  • 컨테이너 로그 (대시보드 → 로그 보기)
