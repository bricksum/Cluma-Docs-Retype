# 클러스터 모니터링

Grafana 기반 클러스터 실시간 모니터링 접근 방법을 설명합니다.

---

## 개요

클러스터 모니터링은 Grafana 대시보드를 통해 CPU, 메모리, GPU 등 클러스터 리소스의 시계열 사용량을 확인하는 기능입니다. 사이드바의 **클러스터 모니터링**을 클릭하면 Grafana가 새 탭으로 열립니다.

Grafana는 Cluma와 별도로 Kubernetes 클러스터 내 `monitoring` 네임스페이스에 배포되며, Cluma 관리자가 해당 URL을 설정한 경우에만 접근할 수 있습니다.

!!!note 버튼 클릭 시 아무 반응이 없는 경우
Grafana URL이 설정되지 않은 상태입니다. 클러스터 배포 담당자에게 `NEXT_PUBLIC_GRAFANA_URL` 환경변수 설정을 요청하세요.
!!!

---

## 접근 방법

관리자 메뉴 → **클러스터 모니터링** 클릭 → Grafana 대시보드가 새 탭으로 열립니다.

---

## Grafana에서 확인할 수 있는 정보

Grafana 대시보드에서는 클러스터의 리소스 사용 추이를 시계열 그래프로 확인할 수 있습니다.

| 항목 | 설명 |
|------|------|
| CPU 사용률 | 노드별·전체 CPU 사용률 추이 |
| 메모리 사용률 | 노드별·전체 메모리 사용률 추이 |
| GPU 사용률 | GPU 노드의 가속기 사용률 추이 |
| 네트워크 I/O | 노드별 네트워크 입출력 트래픽 |
| 디스크 I/O | 노드별 디스크 읽기/쓰기 트래픽 |
| Pod 상태 | 네임스페이스별 Pod 수 및 상태 추이 |

---

## Grafana 접속 정보

Grafana는 클러스터 배포 시 설정된 주소로 접근합니다. 기본 설정은 아래와 같습니다.

| 항목 | 기본값 |
|------|--------|
| Grafana 주소 | `http://grafana.internal` |
| 네임스페이스 | `monitoring` |

!!!note 내부 도메인 접근 설정
`grafana.internal` 등 내부 도메인을 사용하는 경우, 클라이언트 PC의 `/etc/hosts` 파일에 아래 항목을 추가해야 합니다.

```
<노드 IP>  grafana.internal
```
!!!

---

## 클러스터 대시보드와의 차이

| | 클러스터 대시보드 | 클러스터 모니터링 |
|---|---|---|
| **목적** | 노드 상태 및 Pod 현황 실시간 확인 | 리소스 사용량 시계열 분석 |
| **데이터** | 현재 상태 스냅샷 | 과거~현재 추이 그래프 |
| **도구** | Cluma 내장 | Grafana (외부 연동) |
