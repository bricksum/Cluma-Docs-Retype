# 유저 / 그룹 관리

유저 생성·수정·삭제, 그룹 생성·리소스 한도 설정, 유저-그룹 할당 방법을 설명합니다.

---
 
## 유저 관리
 
### 유저 생성
 
관리자 메뉴 → **유저 관리** → **유저 추가**에서 새로운 사용자를 생성합니다.
 
| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| 이름 | 사용자 표시 이름 | 필수 |
| 이메일 | 로그인에 사용할 이메일 주소 | 필수 |
| 비밀번호 | 초기 비밀번호 설정 | 필수 |
| 역할 | 관리자(Admin) 또는 일반 사용자(User) | 필수 |
 
신규 유저는 생성 시 **Default 그룹**에 자동으로 배정됩니다. 필요에 따라 추가 그룹을 할당할 수 있습니다.
 
### 유저 수정
 
유저 목록에서 대상 유저를 선택한 후 이름, 이메일, 역할을 수정할 수 있습니다.
 
### 유저 활성화 / 비활성화
 
계정을 일시적으로 사용 중지할 수 있습니다.
 
- **비활성화**: 해당 유저는 로그인이 불가능해지며, 실행 중인 컨테이너는 자동으로 정지됩니다.
- **활성화**: 비활성화된 계정을 다시 사용 가능한 상태로 전환합니다.
 
!!!warning 유저 비활성화 시 주의
비활성화 시 해당 유저의 실행 중인 컨테이너가 정지됩니다. 중요한 작업이 진행 중인지 사전에 확인하세요.
!!!
 
### 유저 삭제
 
유저를 영구적으로 삭제합니다.
 
!!!danger 삭제 시 주의
유저를 삭제하면 해당 유저의 컨테이너와 개인 스토리지가 함께 삭제될 수 있습니다. 삭제 전 반드시 확인하세요.
!!!

## 그룹 관리
 
그룹은 유저를 묶고 리소스 한도를 정의하는 단위입니다.
 
### 그룹 생성
 
관리자 메뉴 → **그룹 관리** → **그룹 추가**에서 새로운 그룹을 생성합니다.
 
| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| 그룹 이름 | 그룹 식별 이름 | 필수 |
| 설명 | 그룹 용도 설명 | 선택 |
| CPU 한도 | 그룹 전체에서 사용 가능한 최대 CPU 코어 수 | 필수 |
| 메모리 한도 | 그룹 전체에서 사용 가능한 최대 메모리 (GB) | 필수 |
| GPU 한도 | 그룹 전체에서 사용 가능한 최대 GPU 수 | 필수 |
| 스토리지 한도 | 그룹 전체에서 사용 가능한 최대 스토리지 용량 (GB) | 필수 |
 
### 그룹 리소스 한도 설정
 
그룹에 할당된 리소스 한도는 소속 유저들이 생성할 수 있는 컨테이너 및 스토리지의 총량을 제한합니다.
 
```
예시) AI 연구팀 그룹
├── CPU: 64코어
├── 메모리: 256GB
├── GPU: 8개
└── 스토리지: 2TB
 
→ 소속 유저 A가 GPU 4개 사용 중이면,
   소속 유저 B는 최대 GPU 4개까지만 추가 사용 가능
```
 
유저가 컨테이너를 생성할 때 그룹 리소스 한도를 초과하면 자동으로 생성이 차단되며, 잔여 리소스를 안내하는 메시지가 표시됩니다.
 
### 그룹 수정
 
기존 그룹의 이름, 설명, 리소스 한도를 수정할 수 있습니다.
 
!!!warning 리소스 한도 축소 시 주의
리소스 한도를 현재 사용량보다 낮게 설정할 경우, 기존 컨테이너는 유지되지만 새로운 컨테이너 생성이 제한됩니다.
!!!
 
### 그룹 삭제
 
그룹을 삭제하면 그룹 내 모든 리소스(컨테이너, 스토리지)가 함께 삭제됩니다.
 
!!!danger 그룹 삭제 시 주의
그룹 삭제는 되돌릴 수 없습니다. 그룹에 소속된 모든 컨테이너와 스토리지가 영구 삭제되므로 신중하게 진행하세요.
!!!

## 유저-그룹 할당
 
### 유저를 그룹에 추가
 
관리자 메뉴 → **그룹 관리** → 대상 그룹 선택 → **멤버 관리**에서 유저를 추가합니다.
 
- 한 유저는 여러 그룹에 동시에 소속될 수 있습니다.
- 유저가 여러 그룹에 소속된 경우, 컨테이너 생성 시 소속 그룹 중 하나를 선택합니다.
 
### 유저를 그룹에서 제거
 
그룹의 멤버 관리에서 유저를 제거합니다. 유저가 마지막 그룹에서 제거되면 리소스를 사용할 수 없게 되므로, 최소 하나의 그룹에 소속된 상태를 유지하는 것을 권장합니다.

## GPU 사용 시간 관리
 
그룹 또는 유저 단위로 GPU 사용 시간을 할당하고 관리할 수 있습니다.
 
| 설정 단위 | 설명 |
|-----------|------|
| 그룹 단위 | 그룹 전체 유저의 GPU 사용 시간 총합 제한 |
| 유저 단위 | 개별 유저의 GPU 사용 시간 제한 |
 
할당된 GPU 사용 시간이 소진되면 실행 중인 GPU 컨테이너가 자동으로 정지되고, 새로운 GPU 컨테이너 생성이 제한됩니다. 관리자는 언제든지 GPU 사용 시간을 추가 할당할 수 있습니다.
