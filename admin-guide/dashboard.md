# 관리자 대시보드

전체 유저·그룹의 리소스 사용량을 통합 모니터링하는 방법을 설명합니다.

---


## 개요
 
관리자 대시보드는 클러스터 전체의 리소스 사용 현황을 한눈에 파악할 수 있는 통합 모니터링 화면입니다. 일반 사용자의 개인 대시보드와 달리, 모든 유저와 그룹의 리소스를 종합적으로 확인할 수 있습니다.
 
---
 
## 클러스터 리소스 현황
 
대시보드 상단에서 클러스터 전체의 리소스 사용량을 확인합니다.
 
| 항목 | 설명 |
|------|------|
| CPU | 전체 CPU 코어 중 사용 중인 코어 수 및 사용률 |
| 메모리 | 전체 메모리 중 사용 중인 용량 및 사용률 |
| GPU | 전체 GPU 중 할당된 GPU 수 및 사용률 |
| 스토리지 | 전체 스토리지 용량 중 사용 중인 용량 및 사용률 |
| 컨테이너 | 실행 중인 컨테이너 수 및 상태별 분포 |
| 노드 | 전체 노드 수 및 상태별 분포 |
 
---
 
## 그룹별 리소스 사용 현황
 
각 그룹이 할당받은 리소스 한도 대비 실제 사용량을 확인할 수 있습니다.
 
### 확인 가능한 정보
 
- 그룹별 CPU, 메모리, GPU, 스토리지 사용량 / 한도
- 그룹별 실행 중인 컨테이너 수
- 그룹별 소속 유저 수
- 그룹 리소스 사용률 (한도 대비 %)
 
### 필터링
 
- 특정 그룹을 선택하여 상세 사용 현황 확인
- 리소스 사용률 기준으로 정렬
- 리소스 한도에 근접한 그룹 우선 표시
 
---
 
## 유저별 리소스 사용 현황
 
개별 유저의 리소스 사용량을 확인할 수 있습니다.
 
### 확인 가능한 정보
 
- 유저별 CPU, 메모리, GPU, 스토리지 사용량
- 유저별 실행 중인 컨테이너 수 및 상태
- 유저별 GPU 사용 시간 잔여량 (시간 제한이 설정된 경우)
- 유저 소속 그룹 정보
 
### 필터링
 
- 특정 그룹 소속 유저만 표시
- 리소스 사용량 기준으로 정렬
- 활성/비활성 유저 필터
 
---
 
## 컨테이너 상태 분포
 
클러스터에서 실행 중인 모든 컨테이너의 상태를 한눈에 파악할 수 있습니다.
 
| 상태 | 색상 | 설명 |
|------|------|------|
| Creating | 파란색 | 생성 중 |
| Pending | 연한 파란색 | 리소스 대기 중 |
| Running | 녹색 | 정상 실행 중 |
| Deleting | 노란색 | 삭제 진행 중 |
| Stopped | 회색 | 정지됨 |
| Failed | 빨간색 | 오류 발생 |
| Completed | 보라색 | 작업 완료 |
 
---
 
## 노드 상태 현황
 
클러스터에 등록된 모든 노드의 상태와 리소스 사용량을 확인합니다.
 
### 노드별 확인 가능한 정보
 
- 노드 이름 및 IP 주소
- 노드 상태 (Ready / NotReady / SchedulingDisabled)
- CPU / 메모리 사용률
- GPU 사용률 (GPU 노드인 경우)
- 실행 중인 컨테이너 수
