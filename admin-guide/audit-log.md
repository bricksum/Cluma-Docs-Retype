# 감사 로그

모든 유저·관리자 액션에 대한 이력 기록 및 추적 방법을 설명합니다.

---

 
## 개요
 
Cluma는 플랫폼에서 발생하는 모든 주요 액션을 감사 로그로 기록합니다. 관리자는 감사 로그를 통해 누가, 언제, 어떤 작업을 수행했는지 추적할 수 있습니다.
 
---
 
## 감사 로그 조회
 
관리자 메뉴 → **감사 로그**에서 전체 액션 이력을 조회할 수 있습니다.
 
### 로그에 포함되는 정보
 
| 항목 | 설명 |
|------|------|
| 시간 | 액션이 발생한 날짜와 시간 |
| 수행자 | 액션을 수행한 유저 이름 및 역할 |
| 액션 유형 | 생성, 수정, 삭제, 상태 변경 등 |
| 대상 리소스 | 액션이 적용된 리소스 (유저, 그룹, 컨테이너 등) |
| 상세 내용 | 변경 전/후 값 등 구체적인 변경 사항 |
| 결과 | 성공 또는 실패 |
 
---
 
## 기록되는 액션 분류
 
### 유저 관련
 
- 유저 생성, 정보 수정, 활성화/비활성화, 삭제
- 로그인 / 로그아웃
 
### 그룹 관련
 
- 그룹 생성, 정보 수정, 리소스 한도 변경, 삭제
- 유저-그룹 할당 / 제거
 
### 컨테이너 관련
 
- 컨테이너 생성, 이름 변경, 상태 변경(시작/정지), 삭제
- 포트 포워딩 추가 / 삭제
 
### 스토리지 관련
 
- 스토리지 생성, 용량 변경, 삭제
- 스토리지 마운트 / 해제
 
### 노드 관련
 
- 노드 등록, 정보 수정, 활성화/비활성화
- MIG 설정 변경
 
### 레지스트리 관련
 
- 레지스트리 등록, 정보 수정, 삭제
 
---
 
## 필터링 및 검색
 
### 필터 옵션
 
| 필터 | 설명 |
|------|------|
| 기간 | 시작일 ~ 종료일 지정 |
| 수행자 | 특정 유저의 액션만 조회 |
| 액션 유형 | 생성, 수정, 삭제 등 특정 유형만 조회 |
| 대상 리소스 | 유저, 그룹, 컨테이너, 스토리지, 노드 등 리소스 유형별 조회 |
| 결과 | 성공 또는 실패만 조회 |
 
키워드 검색을 통해 특정 리소스 이름, 유저 이름 등으로 로그를 검색할 수도 있습니다.
 
---
 
## 활용 사례
 
### 보안 감사
 
- 비정상적인 시간대의 관리자 액션 확인
- 특정 유저의 대량 리소스 생성/삭제 패턴 파악
- 로그인 실패 이력 추적
 
### 문제 원인 파악
 
- 컨테이너가 예기치 않게 삭제된 경우, 누가 삭제했는지 확인
- 그룹 리소스 한도가 변경된 시점과 변경자 추적
- 노드 비활성화로 인한 서비스 영향 범위 파악
 
### 운영 리포트
 
- 기간별 컨테이너 생성/삭제 현황 파악
- 그룹별 리소스 사용 트렌드 분석
- 유저별 GPU 사용 패턴 확인
