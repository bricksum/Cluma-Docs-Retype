# 노드 관리

클러스터 노드 등록·수정·활성화, GPU 모델·MIG 설정 방법을 설명합니다.

---
 
## 개요
 
노드는 컨테이너가 실행되는 물리적 서버입니다. 관리자는 클러스터에 포함된 노드를 등록하고, 각 노드의 역할·리소스·GPU 정보를 관리할 수 있습니다.
 
---
 
## 노드 등록
 
관리자 메뉴 → **노드 관리** → **노드 추가**에서 새로운 노드를 등록합니다.
 
### 등록 항목
 
| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| 노드 이름 | 노드 식별 이름 | 필수 |
| IP 주소 | 노드의 네트워크 주소 | 필수 |
| 노드 종류 | CPU 전용 노드 또는 GPU 노드 | 필수 |
| 역할 | Data Plane(워커) 또는 Control Plane(마스터) | 필수 |
| CPU 코어 수 | 노드에 탑재된 총 CPU 코어 | 필수 |
| 메모리 용량 | 노드에 탑재된 총 메모리 (GB) | 필수 |
| GPU 모델 | GPU 모델명 (예: A100, V100, H100) | GPU 노드 시 필수 |
| GPU 수량 | 탑재된 GPU 수 | GPU 노드 시 필수 |
 
### 노드 종류
 
| 종류 | 설명 |
|------|------|
| **CPU 노드** | GPU가 없는 일반 컴퓨팅 노드. CPU 기반 작업에 사용 |
| **GPU 노드** | NVIDIA GPU가 탑재된 노드. AI/ML 학습 및 추론에 사용 |
 
---
 
## 노드 수정
 
노드 목록에서 대상 노드를 선택하여 노드 이름, IP 주소, 노드 종류, 역할, 리소스 사양, GPU 정보를 수정할 수 있습니다.
 
!!!warning IP 주소 변경 시 주의
노드 IP 주소가 변경되면, 해당 노드에서 실행 중인 컨테이너의 접속 정보가 변경됩니다. 사용자에게 사전 안내가 필요합니다.
!!!
 
---
 
## 노드 활성화 / 비활성화
 
| 상태 | 설명 |
|------|------|
| **활성** | 새로운 컨테이너가 배치될 수 있는 정상 상태 |
| **비활성** | 새로운 컨테이너 배치 차단. 기존 실행 중인 컨테이너는 유지 |
 
### 비활성화가 필요한 경우
 
- 노드 하드웨어 유지보수 예정
- GPU 드라이버 업데이트
- MIG 설정 변경
 
### 비활성화 절차
 
1. 노드 목록에서 대상 노드 선택
2. **비활성화** 클릭
3. 해당 노드에 새로운 컨테이너 배치가 차단됨
4. 기존 실행 중인 컨테이너는 계속 동작
5. 유지보수 완료 후 **활성화** 클릭하여 복구
 
---
 
## 노드 상태 확인
 
노드 관리 페이지에서 각 노드의 상태를 확인할 수 있습니다.
 
| 상태 | 색상 | 설명 |
|------|------|------|
| Ready | 녹색 | 정상 동작 중 |
| NotReady | 빨간색 | 노드 장애 또는 통신 불가. 즉시 확인 필요 |
| SchedulingDisabled | 회색 | 관리자에 의해 비활성화됨 |
| Unknown | 회색 | 상태를 확인할 수 없음 |

## GPU 분할 사용 (MIG)
 
NVIDIA Multi-Instance GPU(MIG) 기술을 통해 하나의 물리 GPU를 여러 독립적인 GPU 인스턴스로 분할하여 사용할 수 있습니다. 각 인스턴스는 고유한 메모리와 컴퓨팅 리소스를 가지며, 서로 완전히 격리됩니다.
 
MIG는 A100, A30, H100 등 지원 GPU에서 사용 가능하며, 하나의 GPU를 최대 7개의 독립 인스턴스로 분할할 수 있습니다.
 
### MIG 설정 절차
 
1. 대상 GPU 노드를 **비활성화**
2. 해당 노드에서 실행 중인 모든 GPU 컨테이너를 정지
3. 노드 관리 → 대상 노드 선택 → **MIG 설정**
4. GPU별 분할 프로파일 선택
5. 설정 적용 후 노드를 다시 **활성화**
 
!!!warning MIG 설정 변경 시 주의
MIG 프로파일을 변경하려면 해당 GPU를 사용하는 모든 컨테이너를 먼저 정지해야 합니다. 사용자에게 사전 공지 후 진행하세요.
!!!
 
### MIG 활용 예시
 
```
A100 80GB 1대를 분할:
 
분할 전: 1명이 전체 GPU 사용
 
분할 후:
├── 10GB 인스턴스 × 4개 → 추론/테스트용 (4명 동시 사용)
└── 20GB 인스턴스 × 1개 → 소규모 학습용
 
→ 동일한 물리 GPU로 더 많은 유저가 동시에 작업 가능
```
