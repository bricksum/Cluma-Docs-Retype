# 레지스트리 관리

컨테이너 이미지 레지스트리 등록 및 관리 방법을 설명합니다.

---
 
## 개요
 
Cluma에서 컨테이너를 생성할 때 사용하는 이미지는 레지스트리에서 가져옵니다. 관리자는 레지스트리를 등록하여, 사용자들이 컨테이너 생성 시 이미지를 탐색하고 선택할 수 있도록 합니다.
 
---
 
## 레지스트리 등록
 
관리자 메뉴 → **레지스트리 관리** → **레지스트리 추가**에서 새로운 이미지 레지스트리를 등록합니다.
 
### 등록 항목
 
| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| 레지스트리 이름 | 레지스트리 식별 이름 | 필수 |
| URL | 레지스트리 접속 주소 | 필수 |
| 인증 방식 | 인증 없음 / 사용자명+비밀번호 / 토큰 | 필수 |
| 사용자명 | 인증이 필요한 경우 접속 사용자명 | 조건부 |
| 비밀번호 / 토큰 | 인증이 필요한 경우 접속 비밀번호 또는 토큰 | 조건부 |
| 설명 | 레지스트리 용도 설명 | 선택 |
 
---
 
## 레지스트리 수정
 
등록된 레지스트리의 URL, 인증 정보, 설명 등을 수정할 수 있습니다.
 
!!!warning 인증 정보 변경 시 주의
레지스트리 인증 정보가 변경되면, 해당 레지스트리에서 이미지를 사용하는 신규 컨테이너 생성에 영향을 줄 수 있습니다. 변경 전 접속 테스트를 권장합니다.
!!!
 
---
 
## 레지스트리 삭제
 
더 이상 사용하지 않는 레지스트리를 삭제합니다.
 
- 삭제된 레지스트리의 이미지를 사용 중인 기존 컨테이너는 계속 동작합니다.
- 단, 해당 레지스트리의 이미지로 새로운 컨테이너를 생성할 수 없게 됩니다.
 
---
 
## 이미지 탐색
 
등록된 레지스트리의 이미지 목록을 탐색할 수 있습니다.
 
### 탐색 가능한 정보
 
- 이미지 이름
- 태그 목록 (latest, v1.0, cuda-12.0 등)
- 이미지 크기
- 마지막 업데이트 시간
 
---
 
## 사용자의 이미지 사용 방식
 
일반 사용자가 컨테이너를 생성할 때 이미지를 선택하는 방식은 두 가지입니다.
 
1. **레지스트리 탐색**: 관리자가 등록한 레지스트리에서 이미지를 탐색하고 선택
2. **직접 입력**: 퍼블릭 이미지 주소를 직접 입력
 
---
 
## 사용자 커스텀 이미지
 
일반 사용자도 자신만의 커스텀 이미지를 등록하고 관리할 수 있습니다. 관리자는 사용자 또는 그룹별로 이미지 저장 용량 정책을 설정할 수 있으며, 할당된 용량을 초과하면 새로운 이미지 등록이 제한됩니다.
