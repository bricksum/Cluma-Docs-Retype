# 레포지토리 관리

컨테이너 이미지를 가져올 레포지토리 등록 및 관리 방법을 설명합니다.

---

## 개요

Cluma에서 컨테이너를 생성할 때 사용하는 이미지는 레포지토리(이미지 저장 서버)에서 가져옵니다. 관리자는 레포지토리를 등록하여, 사용자들이 컨테이너 생성 시 이미지를 탐색하고 선택할 수 있도록 합니다.

Harbor, Nexus, GitLab Container Registry 등 Docker Registry 프로토콜을 지원하는 서버라면 등록할 수 있습니다.

---

## 레포지토리 등록

관리자 메뉴 → **레포지토리 관리** → **+ 등록하기**에서 새로운 레포지토리를 등록합니다.

### 등록 항목

| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| 레포지토리 이름 | 레포지토리 식별 이름 (예: my-repository) | 필수 |
| 레포지토리 주소 | 이미지 서버 접속 주소 (예: http://192.168.21.29) | 필수 |

!!!note 주소 형식
`http://` 또는 `https://`를 포함한 전체 URL을 입력합니다.
내부 네트워크 주소(사설 IP)도 사용 가능합니다.
!!!

---

## 레포지토리 수정

등록된 레포지토리의 이름 또는 주소를 수정할 수 있습니다.

레포지토리 목록에서 대상 항목의 수정 아이콘을 클릭합니다.

---

## 레포지토리 삭제

더 이상 사용하지 않는 레포지토리를 삭제합니다.

- 삭제된 레포지토리의 이미지를 사용 중인 기존 컨테이너는 계속 동작합니다.
- 단, 해당 레포지토리의 이미지로 새로운 컨테이너를 생성할 수 없게 됩니다.

---

## 이미지 탐색

등록된 레포지토리의 이미지 목록을 탐색할 수 있습니다.

컨테이너 생성 시 이미지 선택 단계에서 등록된 레포지토리를 선택하면 이미지 목록을 바로 탐색할 수 있습니다.

### 탐색 가능한 정보

- 이미지 이름
- 태그 목록 (latest, v1.0, cuda-12.0 등)

---

## 사용자의 이미지 사용 방식

일반 사용자가 컨테이너를 생성할 때 이미지를 선택하는 방식은 두 가지입니다.

1. **레포지토리 탐색**: 관리자가 등록한 레포지토리에서 이미지를 탐색하고 선택
2. **직접 입력**: 퍼블릭 이미지 주소를 직접 입력 (예: `pytorch/pytorch:2.1.0-cuda11.8-cudnn8-runtime`)
