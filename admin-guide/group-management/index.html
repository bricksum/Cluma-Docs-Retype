그룹 관리

그룹 생성·수정·삭제 및 리소스 한도 설정 방법을 설명합니다.

개요

그룹은 사용자를 묶고 리소스 한도를 정의하는 단위입니다. 사용자는 반드시 하나 이상의 그룹에 소속되어야 컨테이너와 스토리지를 생성할 수 있습니다.

그룹 목록

관리자 메뉴 → 그룹 관리에서 전체 그룹 목록을 확인할 수 있습니다.

컬럼 설명
그룹명 그룹 식별 이름
CPU 할당된 CPU 코어 수
메모리(GB) 할당된 메모리 용량
디바이스 할당된 GPU 등 디바이스 수
스토리지(GB) 할당된 스토리지 용량
멤버수 소속 사용자 수
작업 수정(연필 아이콘) / 삭제(휴지통 아이콘)

그룹 생성

우측 상단 그룹 추가하기 버튼을 클릭합니다.

그룹명

그룹을 식별할 이름을 입력합니다. (예: 개발팀, 연구팀)

리소스 할당

그룹에 할당할 리소스를 설정합니다. 화면 상단에 할당 가능한 클러스터 잔여 리소스가 실시간으로 표시되므로, 이를 참고하여 설정합니다.

항목 단위 설명
CPU 코어 2의 배수 단위 선택 그룹 전체에서 사용 가능한 최대 CPU 코어 수
메모리 2의 지수 단위 선택 (2, 4, 8, 16, 32 GB...) 그룹 전체에서 사용 가능한 최대 메모리
스토리지 10GB 단위 선택 또는 직접 입력 그룹 전체에서 사용 가능한 최대 스토리지 용량

디바이스 할당

클러스터에 등록된 디바이스(NVIDIA GPU 등) 유형별로 할당 수량을 설정합니다. 디바이스 카드에 배분 가능 수 / 전체 수가 표시됩니다.

할당 후 남은 리소스

설정 완료 전, 하단의 할당 후 남은 리소스 섹션에서 이 그룹에 할당하고 나면 클러스터에 얼마나 남는지 미리 확인할 수 있습니다.

예시) Default 그룹 설정
├── CPU: 20 코어
├── 메모리: 64 GB
├── 스토리지: 90 GB
└── NVIDIA GPU: 1 개

→ 소속 사용자 A가 GPU 1개 사용 중이면,
   소속 사용자 B는 GPU를 추가 사용할 수 없음
리소스 한도 초과 방지

그룹 리소스 한도를 초과하면 소속 사용자가 컨테이너를 생성할 수 없습니다. 대시보드에서 현재 그룹의 잔여 리소스를 확인하여 적절히 할당하세요.

그룹 수정

목록에서 수정할 그룹의 연필 아이콘을 클릭합니다.

그룹명과 리소스 할당량(CPU, 메모리, 스토리지, 디바이스)을 수정할 수 있습니다. 수정 화면에서도 현재 클러스터 잔여 리소스와 할당 후 남은 리소스를 실시간으로 확인할 수 있습니다.

리소스 한도 축소 시 주의

리소스 한도를 현재 사용량보다 낮게 설정할 경우, 기존 컨테이너는 유지되지만 새로운 컨테이너 생성이 제한됩니다.

그룹 삭제

목록에서 삭제할 그룹의 휴지통 아이콘을 클릭합니다. 삭제 확인을 위해 그룹 이름을 직접 입력해야 합니다.

그룹 삭제 시 주의

이 작업은 되돌릴 수 없습니다. 그룹을 삭제하면 그룹에 소속된 모든 컨테이너와 스토리지가 영구 삭제됩니다.

사용자-그룹 할당

사용자를 그룹에 추가

사용자 생성 시 그룹을 지정하거나, 관리자 메뉴 → 사용자 관리 → 사용자 수정에서 그룹을 추가할 수 있습니다.

  • 한 사용자는 여러 그룹에 동시에 소속될 수 있습니다.
  • 여러 그룹에 소속된 경우, 컨테이너 생성 시 소속 그룹 중 하나를 선택합니다.

사용자를 그룹에서 제거

사용자 수정 화면에서 그룹을 제거합니다. 사용자가 모든 그룹에서 제거되면 리소스를 사용할 수 없게 되므로, 최소 하나의 그룹에 소속된 상태를 유지하는 것을 권장합니다.
