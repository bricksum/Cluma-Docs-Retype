# 컨테이너 관리

Cluma에서 GPU 개발 환경(컨테이너)을 생성하고 접속하는 방법을 안내합니다.

---

## 컨테이너 목록

사이드바에서 **컨테이너**를 선택하면 내가 생성한 컨테이너 목록이 표시됩니다.

| 열 | 설명 |
|------|------|
| 이름 | 컨테이너 이름 (클릭 시 상세 페이지 이동) |
| 상태 | 현재 실행 상태 (Creating / Pending / Running / Deleting / Stopped / Failed 등) |
| 사용자 | 생성한 사용자 |
| 그룹 | 소속 그룹 |
| 작업 | Jupyter / Terminal / VSCode / 삭제 |

**상태 / 사용자 / 그룹** 열은 클릭 시 필터 드롭다운이 표시됩니다. **이름** 열은 클릭 시 오름차순/내림차순 정렬이 가능합니다.

### 목록에서 바로 접속

| 아이콘 | 동작 |
|------|------|
| Jupyter | JupyterLab을 브라우저 새 탭으로 열기 (jupyter 포트 필요) |
| Terminal | 시스템 터미널 앱으로 SSH 연결 시도 + SSH 명령어 클립보드 자동 복사 (ssh 포트 필요) |
| VSCode | VSCode Web(code-server)을 브라우저 새 탭으로 열기 (vscode 포트 필요) |
| 삭제 | 컨테이너 삭제 |

---

## 컨테이너 생성

**새 컨테이너 만들기** 버튼을 클릭하면 생성 모드를 선택할 수 있습니다.

### 원클릭 모드 (템플릿 선택)

저장된 템플릿을 선택하면 이미지, 리소스, 포트, 스토리지 등 설정이 자동으로 채워집니다.

1. 템플릿 카드에서 원하는 템플릿을 선택합니다. 카드에는 이미지, CPU/메모리 설정이 표시됩니다.
2. 확인 모달에서 컨테이너 이름을 입력합니다.
3. **생성** 버튼을 클릭합니다.

!!!note 템플릿 삭제
확인 모달에서 휴지통 아이콘을 클릭하면 해당 템플릿을 삭제할 수 있습니다.
!!!

### 고급 모드 (직접 설정)

항목을 직접 입력하여 컨테이너를 생성합니다.

#### 기본 정보

| 항목 | 설명 |
|------|------|
| 컨테이너 이름 | 컨테이너 식별 이름 |

#### 이미지

이미지 선택 방식을 라디오 버튼으로 선택합니다.

| 방식 | 설명 |
|------|------|
| 직접 입력하기 | Docker Hub 등 퍼블릭 이미지 주소를 직접 입력 |
| 레지스트리에서 이미지 선택 | 관리자가 등록한 Private Registry에서 탐색하여 선택 |
| 커스텀 이미지 | 사용자가 직접 등록한 커스텀 이미지에서 선택 |

#### 리소스

컨테이너에 할당할 컴퓨팅 리소스를 설정합니다.

| 항목 | 선택 가능한 값 |
|------|------|
| CPU | 2, 4, 6, 8, 10, 12 코어 |
| 메모리 | 1, 2, 4, 8, 16, 32 GB |
| 가속기(GPU 등) | 클러스터에 등록된 디바이스 유형과 최대 수량 표시, 0이면 CPU 전용 |

**NUMA-aware CPU 할당**

NUMA-aware CPU 할당 토글을 활성화하면, 컨테이너에 할당된 CPU 코어를 동일한 NUMA 노드에서만 선택합니다. 메모리 접근 지연을 줄여 고성능 워크로드의 처리 속도를 향상시킵니다.

!!!warning 리소스 한도 초과
소속 그룹의 리소스 한도를 초과하면 컨테이너를 생성할 수 없습니다.
상단 헤더에서 현재 그룹의 잔여 리소스를 확인하세요.
!!!

#### 포트 포워딩 (선택)

외부에서 컨테이너에 접속할 포트를 설정합니다.

| 도구 | 기본 포트 |
|------|-----------|
| SSH | 22 |
| JupyterLab | 8888 |
| VSCode (code-server) | 8080 |

포트는 생성 후에도 상세 페이지에서 추가하거나 삭제할 수 있습니다.

#### 스토리지 마운트 (선택)

기존에 생성한 스토리지를 컨테이너 내부 경로에 마운트합니다. 스토리지 목록에는 그룹 스토리지 사용 현황(사용 중 / 최대 / 사용 가능)이 함께 표시됩니다.

```
스토리지: my-dataset
마운트 경로: /workspace/data
```

스토리지가 없는 경우 **새 스토리지 생성** 버튼을 클릭하면 생성 모달이 열립니다.

| 항목 | 설명 |
|------|------|
| 이름 | 스토리지 이름 |
| 용량 | 슬라이더로 크기 설정 |
| 스토리지 유형 | 클러스터에 등록된 유형 선택 (예: nfs-csi) |
| 접근성 | Group(그룹 내 공유) 또는 Public(전체 공개) |

#### SSH Key (선택)

프로필에 등록한 SSH Key를 선택하면, 컨테이너에 자동으로 공개키가 주입됩니다.  
SSH 접속 시 비밀번호 없이 Key 기반 인증을 사용할 수 있습니다.

등록된 Key가 없는 경우 **SSH Key 등록** 버튼을 클릭하면 등록 모달이 열립니다.

| 항목 | 설명 |
|------|------|
| 이름 | Key 식별 이름 |
| 공개키 | 직접 입력 또는 파일 업로드 |

#### 환경 변수 (선택)

컨테이너 실행 시 필요한 환경 변수를 추가합니다.  
**이름**과 **값**을 별도 필드에 입력합니다.

#### 실행 명령어 (선택)

컨테이너 시작 시 실행할 커맨드를 지정합니다. 기본 이미지 Entrypoint를 덮어쓸 때 사용합니다.

## 컨테이너 상세

목록에서 컨테이너 이름을 클릭하면 상세 페이지로 이동합니다.

### 컨테이너 정보

| 항목 | 설명 |
|------|------|
| 컨테이너 이름 | 컨테이너 식별 이름 |
| 상태 | 현재 실행 상태 |
| 컨테이너 ID | 내부 식별자 |
| 컨테이너 이미지 | 사용 중인 이미지 |
| 소유자 | 생성한 사용자 |
| 그룹 | 소속 그룹 |
| 마운트된 스토리지 | 마운트된 스토리지 목록 |
| 환경 변수 | 설정된 환경 변수 목록 |

### 컨테이너 연결

#### 빠른 접속

| 버튼 | 동작 |
|------|------|
| Jupyter | JupyterLab을 브라우저 새 탭으로 열기 (jupyter 포트 필요) |
| Terminal | 시스템 터미널 앱으로 SSH 연결 시도 + SSH 명령어 클립보드 자동 복사 (ssh 포트 필요) |
| VSCode | VSCode Desktop을 SSH Remote로 연결 (`vscode://` URL 실행, ssh 포트 필요) |

!!!note VSCode 접속 요구 사항
- VSCode Desktop 앱이 설치되어 있어야 합니다.
- [Remote - SSH](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-ssh) 확장이 설치되어 있어야 합니다.
- SSH 포트가 설정되어 있어야 합니다.
!!!

#### SSH 접속

SSH 포트가 설정된 경우 SSH 접속 명령어가 표시됩니다. 복사 버튼으로 클립보드에 복사할 수 있습니다.

```bash
ssh -p <포트번호> -i $SSH_KEY_PATH root@<노드IP>
```

SSH Key를 등록한 경우 Key 이름과 함께 `$SSH_KEY_PATH` 환경변수 설정 방법이 안내됩니다.

#### 포트포워딩

등록된 포트 목록에는 포트 이름, 외부 접속 주소(`노드IP:포트`), 컨테이너 내부 포트가 표시됩니다.  
각 항목 우측 휴지통 아이콘을 클릭하면 삭제 확인 모달이 열립니다.

!!!warning 포트 삭제 시 주의
포트를 삭제하면 현재 포트포워딩 되어 있는 모든 포트의 세션 연결이 끊깁니다.
!!!

**포트 추가**

하단의 **+ 포트포워딩 추가** 버튼을 클릭하면 추가 모달이 열립니다.

| 항목 | 설명 |
|------|------|
| 포트 이름 | 포트 식별 이름 (예: jupyter, vscode) |
| 컨테이너 포트 | 컨테이너 내부 포트 번호 (1024–65535) |

---

## 컨테이너 삭제

컨테이너 목록의 삭제 아이콘 또는 상세 페이지 **컨테이너 삭제** 버튼을 클릭합니다.

삭제 모달에서 컨테이너 이름을 직접 입력해야 삭제가 활성화됩니다.

!!!danger 삭제 시 주의
컨테이너를 삭제하면 컨테이너 내부 데이터는 모두 사라집니다.  
중요한 데이터는 반드시 마운트된 스토리지에 저장하세요.
!!!

---

## 템플릿으로 저장

자주 사용하는 컨테이너 설정을 템플릿으로 저장해두면, 다음에 원클릭 모드로 동일한 환경을 빠르게 생성할 수 있습니다.

컨테이너 상세 페이지에서 **템플릿으로 저장** 버튼을 클릭하면 저장 모달이 열립니다.

| 항목 | 설명 |
|------|------|
| 이름 | 템플릿 이름 |
| 설명 | 템플릿 설명 |
| 원클릭 자동 생성 용량 | 원클릭 모드로 생성 시 자동 할당할 스토리지 용량 |
| 기존 스토리지 마운트 | 템플릿 적용 시 자동으로 마운트할 스토리지 선택 |

모달 하단에는 저장될 설정(이미지, CPU, 메모리, 가속기, 포트 등)의 미리보기가 표시됩니다.
