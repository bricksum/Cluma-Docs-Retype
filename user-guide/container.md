# 컨테이너 관리

Cluma에서 GPU 개발 환경(컨테이너)을 생성하고 접속하는 방법을 안내합니다.

---

## 컨테이너 생성

### 1단계 - 기본 정보 설정

**컨테이너 이름**과 **이미지**를 설정합니다.

- 이미지는 관리자가 등록한 Registry에서 탐색하거나, 직접 이미지 주소를 입력할 수 있습니다.
- 저장된 **템플릿**이 있다면 선택하여 설정을 자동으로 불러올 수 있습니다.

### 2단계 - 리소스 설정

컨테이너에 할당할 컴퓨팅 리소스를 설정합니다.

| 항목 | 설명 |
|------|------|
| CPU | 코어 수 설정 |
| 메모리 | GB 단위 설정 |
| GPU | 할당할 GPU 수 (0이면 CPU 전용) |
| 할당 방식 | Dynamic(유연) / Static(전용) 선택 |

!!!warning 리소스 한도 초과
소속 그룹의 리소스 한도를 초과하면 컨테이너를 생성할 수 없습니다.
상단 헤더에서 현재 그룹의 잔여 리소스를 확인하세요.
!!!

### 3단계 - 포트 포워딩 설정

외부에서 컨테이너에 접속할 포트를 설정합니다.  
SSH, Jupyter, VSCode 등 필요한 도구에 맞게 포트를 추가하세요.

| 도구 | 기본 포트 |
|------|-----------|
| SSH | 22 |
| JupyterLab | 8888 |
| VSCode (code-server) | 8080 |

포트는 생성 후에도 자유롭게 추가하거나 삭제할 수 있습니다.

### 4단계 - 스토리지 마운트 (선택)

기존에 생성한 스토리지를 컨테이너 내부 경로에 마운트할 수 있습니다.

```
스토리지: my-dataset
마운트 경로: /workspace/data
```

### 5단계 - SSH Key 설정 (선택)

프로필에 등록한 SSH Key를 선택하면, 컨테이너에 자동으로 공개키가 주입됩니다.  
SSH 접속 시 비밀번호 없이 Key 기반 인증을 사용할 수 있습니다.

### 6단계 - 환경 변수 설정 (선택)

컨테이너 실행 시 필요한 환경 변수를 `KEY=VALUE` 형식으로 추가합니다.

---

## 컨테이너 접속

### SSH 접속

```bash
ssh -p <포트번호> root@<노드IP>
```

SSH Key를 등록한 경우:
```bash
ssh -i ~/.ssh/id_rsa -p <포트번호> root@<노드IP>
```

### JupyterLab 접속

브라우저에서 아래 주소로 접속합니다.

```
http://<노드IP>:<포트번호>
```

Cluma 대시보드의 컨테이너 카드에서 **Jupyter 아이콘**을 클릭하면 자동으로 열립니다.

### VSCode (code-server) 접속

브라우저에서 아래 주소로 접속합니다.

```
http://<노드IP>:<포트번호>
```

Cluma 대시보드의 컨테이너 카드에서 **VSCode 아이콘**을 클릭하면 자동으로 열립니다.

## 컨테이너 관리

### 실시간 로그 확인

컨테이너 카드 → **로그 보기**를 클릭하면 컨테이너 로그를 실시간으로 확인할 수 있습니다.

### 컨테이너 이름 변경

컨테이너 카드 → **설정** → **이름 변경**에서 수정할 수 있습니다.

### 컨테이너 삭제

컨테이너 카드 → **삭제**를 클릭합니다.

!!!danger 삭제 시 주의
컨테이너를 삭제하면 컨테이너 내부 데이터는 모두 사라집니다.  
중요한 데이터는 반드시 마운트된 스토리지에 저장하세요.
!!!

### 템플릿으로 저장

자주 사용하는 컨테이너 설정을 템플릿으로 저장해두면, 다음에 동일한 환경을 빠르게 생성할 수 있습니다.

컨테이너 생성 완료 후 → **템플릿으로 저장** 옵션을 선택하세요.

---

## GPU 사용 시간 확인

그룹 또는 개인에게 GPU 사용 시간이 할당된 경우, 잔여 시간을 초과하면 컨테이너가 자동으로 정지되거나 신규 생성이 제한됩니다.

대시보드 상단에서 현재 잔여 GPU 사용 시간을 확인할 수 있습니다.
