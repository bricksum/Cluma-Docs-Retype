# 컨테이너 관리

Cluma에서 GPU 개발 환경(컨테이너)을 생성하고 접속하는 방법을 안내합니다.

---

## 컨테이너 생성

관리자 메뉴 → **컨테이너** → **새 컨테이너 만들기**에서 생성합니다.

컨테이너 생성은 두 가지 모드를 지원합니다.

### 원클릭 모드 (템플릿 선택)

저장된 템플릿을 선택하면 이미지, 리소스, 포트, 스토리지 등 설정이 자동으로 채워집니다. 자주 사용하는 환경을 빠르게 재현할 때 사용합니다.

### 고급 모드 (직접 설정)

항목을 직접 입력하여 컨테이너를 생성합니다.

#### 기본 정보

| 항목 | 설명 |
|------|------|
| 컨테이너 이름 | 컨테이너 식별 이름 |
| 이미지 | 레포지토리 탐색, 퍼블릭 주소 직접 입력, 커스텀 이미지 중 선택 |

#### 리소스

컨테이너에 할당할 컴퓨팅 리소스를 설정합니다.

| 항목 | 설명 |
|------|------|
| CPU | 코어 수 설정 |
| 메모리 | GB 단위 설정 |
| 가속기(GPU 등) | 할당할 가속기 수 (0이면 CPU 전용) |

!!!warning 리소스 한도 초과
소속 그룹의 리소스 한도를 초과하면 컨테이너를 생성할 수 없습니다.
상단 헤더에서 현재 그룹의 잔여 리소스를 확인하세요.
!!!

#### 포트 포워딩 (선택)

외부에서 컨테이너에 접속할 포트를 설정합니다.

| 도구 | 기본 포트 |
|------|-----------|
| SSH | 22 |
| JupyterLab | 8888 |
| VSCode (code-server) | 8080 |

포트는 생성 후에도 상세 페이지에서 추가하거나 삭제할 수 있습니다.

#### 스토리지 마운트 (선택)

기존에 생성한 스토리지를 컨테이너 내부 경로에 마운트합니다.

```
스토리지: my-dataset
마운트 경로: /workspace/data
```

#### SSH Key (선택)

프로필에 등록한 SSH Key를 선택하면, 컨테이너에 자동으로 공개키가 주입됩니다.  
SSH 접속 시 비밀번호 없이 Key 기반 인증을 사용할 수 있습니다.

#### 환경 변수 (선택)

컨테이너 실행 시 필요한 환경 변수를 `KEY=VALUE` 형식으로 추가합니다.

#### 실행 명령어 (선택)

컨테이너 시작 시 실행할 커맨드를 지정합니다. 기본 이미지 Entrypoint를 덮어쓸 때 사용합니다.

---

## 컨테이너 접속

### SSH 접속

컨테이너 상세 페이지에서 **SSH 복사** 버튼을 클릭하면 접속 명령어가 클립보드에 복사됩니다.

```bash
ssh -p <포트번호> -i $SSH_KEY_PATH root@<노드IP>
```

### JupyterLab 접속

컨테이너 상세 페이지에서 **Jupyter 아이콘**을 클릭하면 브라우저 새 탭으로 JupyterLab이 열립니다.

```
http://<노드IP>:<포트번호>
```

### VSCode 접속

컨테이너 상세 페이지에서 **VSCode 아이콘**을 클릭하면 VSCode Desktop이 해당 컨테이너에 SSH Remote로 연결됩니다.

!!!note VSCode 접속 요구 사항
- VSCode Desktop 앱이 설치되어 있어야 합니다.
- [Remote - SSH](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-ssh) 확장이 설치되어 있어야 합니다.
- SSH 포트가 설정되어 있어야 합니다.
!!!

## 컨테이너 관리

### 실시간 로그 확인

컨테이너 상세 페이지에서 컨테이너 로그를 실시간으로 확인할 수 있습니다.

### 포트 추가 / 삭제

컨테이너 상세 페이지에서 포트 포워딩 설정을 추가하거나 삭제할 수 있습니다.

### 컨테이너 삭제

컨테이너 상세 페이지에서 **삭제** 버튼을 클릭합니다.

!!!danger 삭제 시 주의
컨테이너를 삭제하면 컨테이너 내부 데이터는 모두 사라집니다.  
중요한 데이터는 반드시 마운트된 스토리지에 저장하세요.
!!!

### 템플릿으로 저장

자주 사용하는 컨테이너 설정을 템플릿으로 저장해두면, 다음에 원클릭 모드로 동일한 환경을 빠르게 생성할 수 있습니다.
