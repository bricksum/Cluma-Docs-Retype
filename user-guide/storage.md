# 스토리지

스토리지 생성, 마운트, 공유 방법을 설명합니다.

---


## 스토리지 개요
 
Cluma의 스토리지는 Kubernetes PVC(Persistent Volume Claim) 기반으로 동작합니다. 컨테이너를 삭제하더라도 스토리지에 저장된 데이터는 유지됩니다.
 
| 유형 | 설명 |
|------|------|
| 개인 스토리지 | 본인만 접근할 수 있는 전용 저장 공간입니다. |
| 그룹 공유 스토리지 | 동일 그룹 내 유저들이 함께 사용하는 저장 공간입니다. |
 
---
 
## 스토리지 생성
 
1. 사이드바에서 **스토리지**를 선택합니다.
2. **스토리지 생성** 버튼을 클릭합니다.
3. 아래 항목을 설정합니다.
 
| 항목 | 설명 |
|------|------|
| 이름 | 스토리지 이름 |
| 용량 | GB 단위로 설정 |
| StorageClass | 관리자가 등록한 StorageClass 선택 (NFS, Local 등) |
| 공유 여부 | 개인 전용 또는 그룹 공유 선택 |
 
그룹의 스토리지 총 한도를 초과하면 생성할 수 없습니다. 한도 초과가 필요한 경우 관리자에게 문의합니다.
 
---
 
## 컨테이너에 마운트
 
컨테이너 생성 시 4단계(스토리지 마운트)에서 스토리지를 선택하고 마운트 경로를 지정합니다.
 
```
스토리지: my-dataset
마운트 경로: /workspace/data
```
 
하나의 컨테이너에 여러 스토리지를 마운트할 수 있습니다. 이미 실행 중인 컨테이너에는 스토리지를 추가할 수 없으며, 재생성이 필요합니다.
 
---
 
## 그룹 공유 스토리지
 
동일 그룹 내 유저들과 데이터를 공유해야 하는 경우 그룹 공유 스토리지를 생성합니다.
 
- 같은 그룹의 모든 유저가 해당 스토리지를 자신의 컨테이너에 마운트할 수 있습니다.
- 동시에 여러 컨테이너에서 마운트하여 사용할 수 있습니다.
- 데이터 쓰기 충돌에 주의합니다. 동시에 같은 파일을 수정하면 데이터가 손상될 수 있습니다.
 
---
 
## 용량 확장
 
기존 스토리지의 용량을 확장할 수 있습니다.
 
1. 스토리지 목록에서 대상 스토리지를 선택합니다.
2. **용량 확장**을 클릭합니다.
3. 확장할 용량을 입력합니다.
 
그룹의 스토리지 총 한도 내에서만 확장할 수 있습니다.
 
---
 
## 스토리지 삭제
 
스토리지를 삭제하면 저장된 모든 데이터가 영구적으로 삭제됩니다. 삭제 전 필요한 데이터를 백업합니다.
 
컨테이너에 마운트된 상태에서는 삭제할 수 없습니다. 먼저 마운트를 해제한 후 삭제합니다.
