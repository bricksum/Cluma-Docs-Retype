# 스토리지

스토리지 생성, 마운트, 관리 방법을 설명합니다.

---

## 스토리지 개요

Cluma의 스토리지는 Kubernetes PVC(Persistent Volume Claim) 기반으로 동작합니다. 컨테이너를 삭제하더라도 스토리지에 저장된 데이터는 유지됩니다.

---

## 스토리지 목록

사이드바에서 **스토리지**를 선택하면 그룹의 스토리지 목록이 표시됩니다.

| 열 | 설명 |
|------|------|
| 이름 | 스토리지 이름 (클릭 시 상세 페이지 이동) |
| 상태 | 현재 상태 (active / archived / deleted) |
| 사용자 | 생성한 사용자 |
| 그룹 | 소속 그룹 |
| 작업 | 삭제 |

**이름** 열은 오름차순/내림차순 정렬이 가능하며, **상태 / 사용자 / 그룹** 열은 클릭 시 필터 드롭다운이 표시됩니다.

---

## 스토리지 생성

**스토리지 생성** 버튼을 클릭하면 생성 페이지로 이동합니다.

| 항목 | 설명 |
|------|------|
| 스토리지 이름 | 영문, 숫자, `-` 조합, 50자 이하 |
| 스토리지 용량 | 슬라이더 또는 직접 입력 (GB), 그룹 잔여 한도 이하 |
| 스토리지 유형 | 클러스터에 등록된 StorageClass 선택 (예: nfs-csi) |
| 접근성 | Group (그룹 내 공유, 현재 고정값) |

용량 입력란 하단에 **최대 생성 가능 용량**(할당 - 사용 중)이 안내됩니다.

!!!warning 용량 한도 초과
그룹의 스토리지 총 한도를 초과하면 생성할 수 없습니다.
관리자에게 한도 증가를 요청하세요.
!!!

---

## 컨테이너에 마운트

컨테이너 생성 시 **스토리지 마운트** 항목에서 스토리지를 선택하고 마운트 경로를 지정합니다.

```
스토리지: my-dataset
마운트 경로: /workspace/data
```

하나의 컨테이너에 여러 스토리지를 마운트할 수 있습니다.

---

## 스토리지 상세

목록에서 스토리지 이름을 클릭하면 상세 페이지로 이동합니다.

### 스토리지 정보

| 항목 | 설명 |
|------|------|
| 스토리지 이름 | 스토리지 식별 이름 |
| 상태 | 현재 상태 |
| 스토리지 ID | 내부 식별자 |
| 용량 | 설정된 전체 용량 (GB) |
| 스토리지 유형 | 사용 중인 StorageClass |
| 접근성 | group |

**스토리지 사용량** 프로그레스 바에서 현재 사용 중인 용량을 확인할 수 있습니다.

### 스토리지 기능

하단 **스토리지 기능** 카드에서 두 가지 탭을 확인할 수 있습니다.

| 탭 | 설명 |
|------|------|
| 마운트된 컨테이너 | 현재 이 스토리지를 마운트한 컨테이너 목록 |
| 모니터링 | 준비 중 |

---

## 스토리지 삭제

목록의 삭제 아이콘 또는 상세 페이지 **스토리지 삭제** 버튼을 클릭합니다.

삭제 모달에서 스토리지 이름을 직접 입력해야 삭제 버튼이 활성화됩니다.

!!!danger 삭제 시 주의
스토리지를 삭제하면 저장된 모든 데이터가 영구적으로 삭제됩니다.  
삭제 전 필요한 데이터를 반드시 백업하세요.
!!!
